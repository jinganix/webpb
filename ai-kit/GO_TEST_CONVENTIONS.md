# Go test conventions (plugin)

> Read this document before editing tests under `plugin/**`.  
> Java/TS test style: [TEST_CONVENTIONS.md](TEST_CONVENTIONS.md)

---

## 1. Scope

- **Golden tests** (`java_golden_test.go`, `ts_golden_test.go`): verify protoc plugin output matches committed expected files.
- **Unit tests** (`internal/**/**/*_test.go`): parser helpers and isolated logic.

---

## 2. Golden test layout

| Piece | Location |
|-------|----------|
| Proto fixtures | `lib/tests/src/proto/proto2/{case}/`, `lib/tests/src/proto/proto3/{case}/` |
| Serialized dumps | `lib/tests/src/main/resources/{syntax}_{case}/dump/test.dump` (generated by `:lib:tests:build`) |
| Expected Java output | `plugin/testdata/java/{syntax}_{case}/` |
| Expected TS output | `plugin/testdata/ts/{syntax}_{case}/` |
| Test harness | `plugin/internal/testutil/` |

### Fixture cases (`{syntax}_{case}`)

`{syntax}` is `proto2` or `proto3`. `{case}` names a scenario slice:

| `{case}` | Purpose |
|----------|---------|
| `core_codegen` | End-to-end happy path: REST paths, annotations, maps, nested types, imports |
| `generator_options` | File/message/field generator options and edge cases (empty message, `is*` booleans, skipped packages) |
| `alias_skip` | Auto-alias collision avoidance (`a`/`b` → `c`/`d`) |
| `auto_alias` | Auto-alias at webpb / file / message / field levels and extends hierarchy |
| `enumeration` | Const enum defaults and map-valued enums |
| `message_extends` | `extends`, `sub_type`, `sub_values`, cross-file references |
| `imports` | Extends types with empty `java_package` |
| `errors` | Invalid annotations, imports, extends targets, duplicated fields, bad Java imports |

Proto3 fixtures live under `proto3/` and mirror proto2 under `proto2/`. Re-sync proto3 from proto2 with `python3 lib/tests/scripts/convert_proto2_to_proto3.py`.

Run golden tests after building dumps:

```bash
./gradlew :lib:tests:classes :plugin:testGo
# or
cd plugin && go test ./...
```

`testTsGo` runs only `TestTSGolden` when you want a faster TS-only check.

---

## 3. Naming and structure

- Test functions: `Test{Feature}` with subtests via `t.Run("{case}", ...)`.
- Golden mismatch messages should include `{case}/{filename}` for quick navigation.
- Keep golden tests separate from unit tests (`TestJavaGolden` vs `jparse_test.go`).

---

## 4. Updating expected output

1. Fix the generator in `internal/javagen` or `internal/tsgen`.
2. Run golden tests to see diffs.
3. When output is correct, copy updated files into `plugin/testdata/java/` or `plugin/testdata/ts/`.
4. Re-run `go test ./...` until green.

Do not commit scratch diff artifacts (`.golden-got.*`, `.golden-want.*`).

---

## 5. CI

Go plugin tests run via `./gradlew :plugin:testGo` as part of `./gradlew build`.

The TypeScript protoc plugin is published to Maven Central as `webpb-protoc-ts` via `./gradlew :plugin:ts:publish`.
